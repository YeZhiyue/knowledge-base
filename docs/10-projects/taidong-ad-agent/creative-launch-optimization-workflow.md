# 素材创建、投放、优化工作流

更新时间：2026-06-01

## 目标

把广告 Agent 的核心链路拆成可审计、可评测、可逐步自动化的工作流。首阶段不追求全自动投放，而是先让 Agent 生成高质量草案、风险提示和优化建议。

## 端到端流程

```text
1. 输入 brief
2. 结构化解析
3. 素材策略生成
4. 多版本素材创建
5. 合规与品牌检查
6. 投放结构草案
7. 人工确认
8. 平台执行或导出
9. 指标回收
10. 诊断与优化
11. 复盘沉淀
```

## 1. 输入 brief

输入应尽量结构化：

- 产品或服务说明。
- 目标用户。
- 投放平台。
- 地域、语言、预算和时间范围。
- 转化目标。
- 禁用表达和品牌约束。
- 已有素材或历史投放结果。

## 2. 结构化解析

输出：

- 用户画像。
- 搜索意图或问题地图。
- 核心卖点。
- 关键词方向。
- 平台限制。
- 信息缺口。

如果关键信息缺失，Agent 应先提问或生成待补充清单，不应直接进入投放配置。

## 3. 素材创建

素材 Agent 负责生成：

- 标题和描述。
- 关键词广告文案。
- 落地页模块文案。
- 图片或视频脚本。
- 不同人群和不同意图的素材变体。

所有素材都应带上：

- 目标用户。
- 使用场景。
- 主卖点。
- 风险提示。
- 适用平台。

## 4. 合规检查

Compliance Agent 需要检查：

- 平台规则风险。
- 品牌词和禁用表达。
- 夸大承诺。
- 误导性表达。
- 行业敏感限制。
- 落地页和广告文案一致性。

检查结果应输出风险等级、触发原因和可替代表达。

## 5. 投放草案

Launch Agent 只生成草案：

- Campaign / Ad Group 结构。
- 关键词分组。
- 匹配方式建议。
- 创意绑定关系。
- 预算和出价建议。
- 转化追踪需求。
- 人工确认摘要。

涉及真实账户、预算、出价、地域、客户或平台操作的内容，必须经过人工确认。

## 6. 指标诊断

Analyst Agent 根据脱敏报表或公开样例分析：

- 展示量不足。
- CTR 低。
- CVR 低。
- CPA 高。
- 消耗异常。
- 关键词和素材表现分化。

输出应包含诊断证据、可能原因、优先级和需要补充的数据。

## 7. 优化建议

Optimizer Agent 生成可执行建议：

- 新增或暂停关键词。
- 调整匹配方式。
- 替换低效素材。
- 拆分或合并广告组。
- 调整预算分配建议。
- 增加否定关键词。
- 发起 A/B 测试。

所有优化动作都应标记风险等级和是否需要人工审批。

## 8. 复盘沉淀

每轮工作流结束后，沉淀：

- 输入 brief 版本。
- Agent 输出版本。
- 人工修改点。
- 执行结果。
- 指标变化。
- 下轮优化假设。
- 可复用经验。

## 关键原则

- 先做草案和建议，再逐步扩大自动执行范围。
- 先做结构化输出，再追求自然语言丰富度。
- 先做可回放和可审计，再接入真实生产操作。
- 任何高风险动作都必须有人审、权限控制和回滚记录。
