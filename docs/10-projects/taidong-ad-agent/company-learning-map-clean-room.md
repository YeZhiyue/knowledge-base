# 公司学习地图 Clean-Room 版

更新时间：2026-06-01

## 用途

本页用于规划入职前后需要学习和确认的主题，只记录个人抽象后的学习方向和通用问题，不记录 ATA、语雀、内部系统、内部截图、客户信息、账户信息、预算信息、接口细节或内部实现。

## 学习主题地图

| 主题 | 学习目标 |
| --- | --- |
| 产品定位 | 理解两个产品分别解决什么广告问题、服务谁、如何衡量成功 |
| 业务链路 | 理解从客户 brief 到素材、投放、优化、复盘的完整流程 |
| 平台适配 | 理解首批平台、后续平台和多平台抽象方式 |
| Agent 框架 | 理解当前内部试用使用了哪些编排、工具、记忆、评测和审计能力 |
| 素材系统 | 理解素材创建、版本、标签、审核、复用和效果回流 |
| 数据闭环 | 理解报表、指标、转化、实验和优化建议如何联动 |
| 风控合规 | 理解预算、账户、品牌、平台规则和审批边界 |
| 工程协作 | 理解产品、运营、算法、后端、平台工程之间的协作方式 |

## 入职后问题清单

### 产品与业务

- 两个产品分别面向什么用户和使用场景？
- 首阶段最重要的业务指标是什么？
- 当前内部试用已验证哪些价值，未验证哪些假设？

### 广告平台

- 首批接入平台有哪些？
- 平台差异主要集中在账户结构、素材格式、报表指标还是执行 API？
- 哪些平台动作必须人工确认？

### Agent 和工具

- 当前 Agent 能调用哪些工具？
- 工具输入输出是否已有统一 schema？
- Agent 失败时如何回退到人工流程？

### 数据和评测

- 是否已有脱敏样例库？
- 如何评估素材质量、配置质量和优化建议质量？
- 是否支持回放一次 Agent 决策的完整上下文？

### 风控和审计

- 预算、账户、素材、平台规则分别由谁审批？
- 审计日志记录到什么粒度？
- 哪些内部信息不能进入知识库、模型上下文或外部工具？

## 记录边界

可以记录：

- 抽象后的流程图。
- 通用问题清单。
- 个人学习总结。
- 官方公开资料链接。
- 脱敏后的方法论和复盘模板。

不能记录：

- ATA、语雀或内部文档原文。
- 内部系统截图或页面结构。
- 客户名称、账户 ID、预算、合同、真实报表。
- 未公开接口、数据库、模型部署或权限细节。
- 任何密钥、token、cookie、账号密码。
