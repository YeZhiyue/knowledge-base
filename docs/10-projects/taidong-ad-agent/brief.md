# 钛动科技 GEO/SEM 广告 Agent 项目 Brief

更新时间：2026-06-01

## 一句话定义

面向 GEO / SEM 等广告场景，构建一套能覆盖素材创建、投放执行、效果优化全链路，并可扩展到多广告平台的广告 Agent 框架。

## 背景

从原始微信信息看，当前项目已经在内部试用，近期重点是判断哪些 Agent 框架适合目标场景。岗位定位不是纯框架研发，也不是纯业务开发，而是在真实广告业务中持续完善 Agent 框架。

## 目标

### 业务目标

- 支持两个产品快速面向市场。
- 提升广告生产和投放链路效率。
- 形成可跨平台复用的广告 Agent 能力。

### 技术目标

- 建立广告 Agent 的标准任务编排方式。
- 支持素材创建、投放、优化三个核心阶段。
- 支持平台 API、素材库、指标数据、策略规则等工具接入。
- 建立评测、监控、回放和人工兜底机制。

## 范围

### 当前范围

- GEO / SEM 方向。
- Agent 框架调研与选型。
- 知识库、调研记录、决策记录和 Codex Cloud 工作流沉淀。

### 暂不展开

- 真实客户数据接入。
- 生产环境密钥、账号、投放预算等敏感信息。
- 未确认平台的具体 API 开发。

## 核心链路假设

```text
业务目标/产品卖点
  -> 人群与场景理解
  -> 素材策略生成
  -> 素材创建/改写/审核
  -> 投放计划生成
  -> 平台 API 执行或人工确认
  -> 指标回收
  -> 诊断与优化建议
  -> 新一轮素材/投放策略迭代
```

## Agent 能力拆分草案

- Brief Agent：理解广告目标、人群、平台约束和业务卖点。
- Creative Agent：生成标题、落地页文案、关键词、图片/视频脚本等素材方案。
- Compliance Agent：检查平台规则、品牌约束、禁用表达和风险词。
- Launch Agent：生成投放结构、预算建议、定向设置和平台执行计划。
- Analyst Agent：读取数据，诊断 CTR、CVR、CPA、ROI 等指标波动。
- Optimizer Agent：输出关键词、出价、素材、定向、人群包等优化动作。
- Review Agent：对关键动作做质量评估和人工确认前检查。

## 事实 / 判断 / 假设

### 事实

- 项目方向是 GEO / SEM。
- 目标链路包括素材创建、投放、优化。
- 项目已内部试用。

### 判断

- 首阶段更适合从“确定性工作流 + 局部 Agent 自主决策”切入，而不是完全开放式自主 Agent。
- 广告投放动作有预算和业务风险，生产执行必须有人审、权限控制和回滚记录。

### 假设

- 初期最重要的不是模型能力，而是业务上下文、工具封装、评测闭环和可观测性。
- 若项目已经内部试用，已有系统大概率包含部分素材、投放或报表工具，需要先盘点现状。
