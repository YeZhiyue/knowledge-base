# 广告 Agent 总览

更新时间：2026-06-01

## 定位

本知识库服务于 GEO / SEM 广告 Agent 方向的入职前准备和后续项目沉淀。当前重点是把广告业务链路、Agent 框架选型、系统架构、风控评测和学习计划组织成可持续维护的 GitBook 文档站。

## 一句话目标

构建一套可覆盖素材创建、投放执行、效果优化，并可逐步适配多广告平台的广告 Agent 方法论和参考架构。

## 当前事实

- 项目方向聚焦 GEO / SEM。
- 项目本质是广告 Agent。
- 目标链路包括素材创建、投放、优化。
- 框架目标是适配多个广告平台。
- 当前已有内部试用信息，但仓库不记录内部实现、内部截图、客户数据或未脱敏投放数据。

## 初步判断

- 广告 Agent 不应从完全自主执行起步，更适合采用“确定性工作流 + 局部 Agent 推理 + 人工确认”的路线。
- 框架选型需要同时评估业务链路、状态管理、工具治理、结构化输出、可观测性、评测回放和权限审计。
- GEO / SEM 场景要把广告业务知识和 Agent 工程能力一起沉淀，不能只做框架 demo。

## 核心链路

```text
业务目标 / 产品卖点
  -> 用户与搜索意图理解
  -> 关键词 / 主题 / 人群拆解
  -> 素材策略与多版本生成
  -> 平台规则与品牌合规检查
  -> 投放结构和预算草案
  -> 人工确认或审批
  -> 平台执行或导出配置
  -> 指标回收和异常诊断
  -> 优化建议与复盘
```
